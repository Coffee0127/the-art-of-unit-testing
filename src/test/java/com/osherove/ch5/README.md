# 隔離框架的優缺點
## 優點
* 容易驗證參數
* 容易驗證一個方法被多次呼叫
* 容易建立假物件
    * 不需要再手刻模擬物件

## 缺點
* 測試可讀性變差
    * 應考慮去掉一些模擬物件或是期望模擬物件的互動
    * 將測試拆分成幾個更小、更好理解的測試案例
    * 重構測試程式，透過 private method 表達意圖
* 驗證了錯誤的東西
* 一個測試中有多個模擬物件
    * 同時測試多件事情，可能會引起混淆
    * 代表測試單元中的好幾個最終結果
* 過度指定的測試
    * 盡量使用非嚴格模擬物件
    * 盡量使用虛設常式物件，而非模擬物件
        * 避免測試有過多的 Assert
    * 極力避免把虛設常式物件當模擬物件使用
